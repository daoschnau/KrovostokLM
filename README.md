# KrovostokLM 🩸

Telegram-бот «психологической поддержки», который отвечает на любую жизненную ситуацию идеально подобранной цитатой из текстов группы **«Кровосток»**.

Опиши боту свою боль, победу или экзистенциальный тупик — он поймёт суть и ответит хлёстким панчлайном из дискографии Шило и компании, с указанием трека.

```
Пользователь: вечер пятницы, все ушли домой, один я в пустом офисе
              продолжаю пилить свой пет-проджект, который скорее
              всего никому не будет нужен

Бот: 💬 Цитата:
     <суровый, но точный панчлайн из подходящего трека>
     🎵 Трек: dumay_pozitivno
```

## Как это работает

Это RAG-пайплайн (Retrieval-Augmented Generation) с HyDE и реранкингом:

```
Сообщение пользователя
        │
        ▼
[1] HyDE — Claude Haiku формулирует «суровую мораль» по ситуации
        │
        ▼
[2] Векторный поиск — эмбеддинг морали (HF Inference API)
    → ChromaDB → топ-10 похожих цитат из ~840 чанков текстов
        │
        ▼
[3] Реранкинг — Claude Haiku выбирает одну лучшую цитату,
    отсеивая обрывки и буквальные совпадения слов
        │
        ▼
Ответ в Telegram (aiogram)
```

**Почему HyDE.** Запрос пользователя («начальник орёт, хочу уволиться») и тексты песен лежат в разных смысловых пространствах. Поэтому сначала Claude генерирует гипотетический ответ — короткую фаталистичную сентенцию в стилистике корпуса, — и уже по ней ищутся ближайшие цитаты. Совпадение получается смысловым, а не пословным.

**Память диалога.** Бот помнит последние 3 пары «вопрос — ответ» в каждом чате (в оперативной памяти, без БД), так что короткие реплики вроде «и что мне теперь делать?» понимаются в контексте беседы. Команда `/start` сбрасывает историю.

## Стек

| Компонент | Технология |
|---|---|
| Telegram | [aiogram 3](https://docs.aiogram.dev/) (long polling) |
| LLM (HyDE + реранкинг) | Claude Haiku 4.5 ([Anthropic API](https://docs.anthropic.com/)) |
| Эмбеддинги | `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2` через [HF Inference API](https://huggingface.co/docs/inference-providers) — модель не грузится в RAM |
| Векторная база | [ChromaDB](https://www.trychroma.com/) (persistent, лежит в репозитории) |
| Устойчивость | [tenacity](https://tenacity.readthedocs.io/) — ретраи с экспоненциальной паузой |
| Деплой | Docker → [Render](https://render.com/) (Background Worker) |

## Структура проекта

```
KrovostokLM/
├── data/
│   ├── raw/                 # Сырые тексты песен (.txt), один файл — один трек
│   ├── processed/           # dataset.parquet — нарезанные чанки
│   └── vector_db/           # Готовая база ChromaDB (~840 цитат)
├── src/
│   ├── scrape_lyrics.py     # Скрейпер текстов с lyricsworld.ru
│   ├── data_prep.py         # Чанкование: sliding window по 2 строки (двустишия)
│   ├── vectorize.py         # Векторизация чанков в ChromaDB (локальная модель)
│   ├── core.py              # RAG-ядро: HyDE → поиск → реранкинг
│   └── bot.py               # Telegram-бот, точка входа
├── Dockerfile
├── requirements.txt         # Зависимости рантайма (бота)
└── README.md
```

## Запуск

### 1. Переменные окружения

Создайте `.env` в корне проекта (или задайте переменные в окружении):

```env
TG_BOT_TOKEN=...        # токен бота от @BotFather
ANTHROPIC_API_KEY=...   # ключ Anthropic API
HF_TOKEN=...            # токен Hugging Face (тип Read достаточен)
```

### 2. Локально

```bash
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python src/bot.py
```

Готовая векторная база уже лежит в `data/vector_db/`, поэтому бот стартует сразу — пересобирать данные не нужно.

### 3. Docker

```bash
docker build -t krovostoklm .
docker run --env-file .env krovostoklm
```

### 4. Деплой на Render

Репозиторий подключается как **Background Worker** с типом окружения **Docker** (Dockerfile в корне). Переменные `TG_BOT_TOKEN`, `ANTHROPIC_API_KEY`, `HF_TOKEN` задаются во вкладке *Environment*. Auto-Deploy с ветки `main` пересобирает сервис на каждый пуш.

Бот работает через long polling, поэтому открытый порт и вебхуки не нужны — тариф Starter подходит.

## Пересборка базы цитат

Нужна, только если вы меняете корпус текстов. Шаги выполняются локально и требуют дополнительных зависимостей (`pandas`, `pyarrow`, `beautifulsoup4`, `sentence-transformers` — в `requirements.txt` рантайма их нет, чтобы не раздувать контейнер):

```bash
python src/scrape_lyrics.py   # 1. Собрать тексты в data/raw/
python src/data_prep.py       # 2. Нарезать на двустишия → data/processed/dataset.parquet
python src/vectorize.py       # 3. Векторизовать в ChromaDB → data/vector_db/
```

При первом запуске `vectorize.py` скачает модель эмбеддингов (~400 МБ). Готовую базу закоммитьте — в проде она используется как есть.

> **Важно:** модель эмбеддингов в `vectorize.py` (локальная) и в `core.py` (через HF API) должна быть одной и той же, иначе векторы запросов и базы окажутся в разных пространствах и поиск сломается.

## Ограничения

- История диалога живёт в RAM и обнуляется при каждом редеплое/рестарте сервиса.
- База нарезана механически (скользящее окно по 2 строки), поэтому часть чанков — обрывки фраз; их отсев возложен на реранкер.
- Тексты «Кровостока» содержат ненормативную лексику — собственно, в этом и смысл.

## Дисклеймер

Проект сделан с любовью к творчеству группы «Кровосток» в образовательных и развлекательных целях. Права на тексты принадлежат их авторам. Бот не является заменой психологической помощи — хотя иногда работает лучше.
